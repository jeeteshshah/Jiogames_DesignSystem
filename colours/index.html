Usage rules

Hard rules enforced by the DLS validator. Green rows are required practices. Red rows are prohibited — these will cause a validation ERR.

Use --jio for all primary interactive elements — buttons, links, active states, progress bars.
Use --negative for all error states — field errors, destructive confirmations, failure toasts.
Use --text3 only for decorative or non-essential text (captions, metadata) — never for body copy or labels.
Use --ultimate exclusively for All Screen Pass tier UI — do not use for general green accents.
Reference all colour values through their token — var(--jio), never #00A859.
Use blue, indigo, or purple for brand actions — these are not JioGames brand colours.
Use raw hex values in component CSS — every value must reference a token variable.
Use light-mode surfaces (white, off-white, light grey) at any breakpoint on any platform.
Use --text4 for any meaningful, readable content — it is contrast-fail and reserved for disabled states only.
Create new colour values not in the token set — open an RFC instead.
