Empty Search

No-results state for the game search experience. Includes query echo, spelling suggestion, filter reset, and fallback popular content.

Mobile Web TV Beta

Anatomy

The empty search state has 5 layers, each optional depending on context:

1
Query echo Always present. "No results for "[query]"". Query in bold. Max 40 chars shown — truncate with ellipsis.
2
Spelling suggestion Conditional. "Did you mean [correction]?" — tappable. Only shown when fuzzy-match confidence ≥ 0.75.
3
Filter reset Conditional. Show only if filters are active. "Try removing filters" chip. Tapping clears all active filters and re-runs search.
4
Search suggestions Horizontal chip row. 4–6 popular genre/category chips. "Try: Action · Racing · Puzzle". Tap replaces current query.
5
Popular fallback rail Always present. "Popular right now" horizontal card rail (standard game cards, 4–6 items). Personalised if user logged in.

Copy rules

Echo the query, don't apologise
Use: "No results for "clash of—"". Avoid: "Sorry, we couldn't find anything. Please try a different search." (passive, wastes space)
Suggestions are action-labelled
Use: "Try:". Not "Similar searches:" or "You might also like:". One word before the chips.
Fallback rail header
Always "Popular right now". Not "Recommended for you" (that implies personalisation when context may be unrelated).

Platform differences

Mobile

  • Full-screen empty state below search bar
  • Keyboard stays open (user can retype)
  • Fallback rail horizontally scrollable

Web

  • Empty state in results column (right of filters)
  • Filter panel stays visible on left
  • Fallback rail shows as grid (not horizontal scroll)

TV

  • Query echo + suggestions in 48px+ text (TV sizing)
  • Chips focusable via D-pad
  • Fallback rail: initial focus on first card

Accessibility

  • ·Query echo: aria-live="polite" region — announced when results count changes to zero
  • ·Suggestion chips: role="list" + role="listitem". Each chip is a button, not a link.
  • ·Spelling correction: underline style (not colour-only) so visible in forced-colours mode
  • ·Filter reset chip: button with aria-label="Remove all filters and search again"
  • ·Fallback rail: labelled with aria-label="Popular right now" so screen reader announces section
