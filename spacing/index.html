Section 02

Semantic spacing tokens

Semantic aliases resolve to the correct value per platform via media queries. Use the alias — never the scale step directly — for all layout roles listed here.

Token Mobile Web ≥768px TV ≥1280px Usage
--gutter 16px 40px 80px Side margins for all content
--section-gap 32px 48px 64px Gap between page sections / rails
--rail-gap 32px 48px 64px Gap between rail rows
--card-gap 12px 24px 24px Gap between cards in a rail
--component-padding 16px 24px 32px Internal padding for most components
--card-padding 16px 24px 32px Card internal padding
--sheet-padding 24px 32px — Bottom sheet / drawer internal padding
--hero-gap 32px 64px 96px Gap below hero sections
Mobile values are the base (defined in :root). Web and TV values override via media queries in tokens.css. You never need to handle platform differences manually — use the alias and the token resolves automatically.
Section 03

8px base grid

Every spacing value is a multiple of 8px, or a half-step at 4px. This creates consistent vertical rhythm and aligns elements across different screen densities.

The grid below represents an 8×8px cell. Every spacing token aligns to these cells — 1 cell = 8px, 2 cells = 16px, and so on. Half-steps (4px) are permitted only for tight component internals such as icon-to-label gaps.
Each cell = 8×8px. Highlighted cells show 16px (--space-2) increments.

Z-index scale

Use var(--z-*) tokens — never raw integers. Deviating from the scale causes stacking conflicts between nav, modals, toasts, and overlays.

TokenValueUsed by
--z-base0Default in-flow elements
--z-raised10Cards on hover, lifted surfaces
--z-dropdown100Select dropdowns, Combobox, Popover
--z-sticky200Sticky table headers, pinned rails
--z-appbar300AppBar (always above page content)
--z-drawer400Drawer / side panel
--z-modal500Dialog, Bottom Sheet, Command palette
--z-toast600Toast notifications (above modals)
--z-tooltip700Tooltip, Hover Card (always topmost)
--z-tvFocus800TV focus ring overlay
Rule: Toast must sit above modals so subscription confirmation toasts are visible after an in-app purchase dialog closes. Never use z-index: 9999 — it breaks the scale for every layer above it.
Section 04

Usage rules

Enforced by tokens/validate.sh. ERRs block shipping. WARNs must be justified in the QA report.

Use spacing tokens for all layout gaps, margins, and paddings — every spacing value must reference a CSS custom property.
Use --gutter for all side margins — it adapts from 16px (mobile) to 40px (web) to 80px (TV) automatically.
Use 4px (--space-0-5) for sub-8px spacing only in tight component internals such as icon-to-label and badge-to-text gaps.
Use stack utilities (.page-stack, .component-stack, .content-stack) for vertical rhythm — never hand-place margin-bottom on children.
Use --section-gap between rails and major content regions — not --space-4 or a bare 32px value.
Use arbitrary px values like 10px, 15px, 18px, or 20px in new components — these are off the 8px scale and are never approved.
Use rem or em for spacing — all spacing tokens are px; rem introduces font-size coupling that breaks the grid.
Use hardcoded 20px as gutter — use var(--gutter) which adapts per platform. 20px is banned from the spacing scale.
Skip spacing tokens for "one-off" adjustments — every spacing value must use a token or go through the exception approval process.
Use child margin-bottom for vertical rhythm — always use gap on the parent flex/grid container (stack utilities).
