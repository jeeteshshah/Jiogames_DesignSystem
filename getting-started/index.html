What is JioGames DLS?

The JioGames Design Language System is the single source of truth for UI across Mobile, Web, and TV. It defines tokens, components, patterns, and guidelines so every team ships consistent, accessible, premium experiences without reinventing decisions.

475 tokens
Colour, spacing, radius, motion, z-index — all in one CSS file.
53 components
From buttons to rail cards — documented with variants, states, and accessibility.
3 platforms
Mobile, Web, and TV each have their own sizing, spacing, and interaction rules.
Dark-first
Every surface is dark. JioGames green is the sole brand accent — never blue or purple.

Installation

The DLS ships as a static CSS file. No build step, no npm package required for adoption.

1. Import tokens.css

Add this as the first stylesheet on every page. All CSS custom properties are declared on :root.

<link rel="stylesheet" href="/tokens/tokens.css">

2. Import components.css

Component classes live here. Only import what you need — or import the full file and rely on browser CSS opt-in.

<link rel="stylesheet" href="/components.css">

3. Set root styles

html, body {
  background: var(--bg);       /* #06080F dark base */
  color: var(--text);          /* #F4F2EE off-white */
  font-family: var(--jio-font); /* JioType, system fallback */
  margin: 0;
}

4. Platform class (TV only)

On TV surfaces, add .platform-tv to <html> or <body>. This activates TV-specific token overrides for sizing and spacing.

<html class="platform-tv">
JioType font: Font files are served from /Assets/font/. Ensure your project serves these files or configure @font-face to point to your own font CDN. The tokens.css file includes font-display: swap on all weights to prevent FOIT.

Three-tier token model

Never use raw hex or px values in component code. Every value must come from a token.

Tier 1 — Primitive
Raw values
The atomic palette. Never use directly in component code.
--jio: #00A859
--space-2: 16px
--r4: 14px
Tier 2 — Semantic
Purpose-named
Maps a purpose to a primitive. Change the semantic layer; all components update.
--action-primary: var(--jio)
--text-placeholder: #8B92A5
--status-positive: var(--positive)
Tier 3 — Component
Component-scoped
Scoped to one component. Enables component-level theming without touching primitives.
--btn-primary-bg: var(--action-primary)
--input-placeholder: var(--text-placeholder)
--card-radius-landscape: var(--r5)
Rule: always use the most specific tier available
Do background: var(--btn-primary-bg)
Don't background: #00A859
Deprecated token
--ctrl-h → use --input-h
--ctrl-h is deprecated. Canonical input/control height is now --input-h. Exception: --ctrl-h-sm and --ctrl-h-ghost remain valid — they have no --input-h equivalent.

Platform rules overview

The DLS uses the same class names and token names on all three platforms. Platform differences are expressed through token overrides, not separate codebases.

Mobile

  • Touch targets: min 44×44px (--touch-min)
  • Gutter: 16px (--gutter)
  • Bottom tab bar always present
  • Safe area insets via --safe-bot
  • No hover states — use active/pressed only

Web

  • Gutter: 40px (--gutter overridden at 768px)
  • Max container: 1280px (--container-web)
  • Hover states required (--state-hover-bg)
  • Focus ring: 2px jio-green, 2px offset
  • Sidebar navigation instead of tab bar

TV

  • Add .platform-tv to <html>
  • TV safe zone: 80px (--tv-safe)
  • Focus ring: 3px + glow (--tv-focus-shadow)
  • All interactive elements must be D-pad reachable
  • 10-foot test: text must be readable at distance

TV focus gotchas

TV is the hardest platform to get right. These are the most common mistakes.

✕
Rail overflow clips the focus ring
Set overflow: visible on rail containers and use clip-path for visual overflow only.
✕
Focus gets trapped inside a rail
Up/Down arrows must escape the rail. Use tabindex and D-pad navigation management.
✕
Using outline: none without a replacement
Always pair with box-shadow: var(--tv-focus-shadow). Never remove focus visibility. validate.sh catches this.
✕
Forgetting .platform-tv
Without it, TV gets mobile token values. Buttons are too small, focus rings too thin, gutters too tight.
✕
Modal focus trap breaks D-pad exit
Implement Back button / Escape handler explicitly. See Dialogs for the pattern.

Copying code correctly

Code examples in this DLS show the correct HTML structure and class names. A few rules when copying:

Always link tokens.css first
Component classes depend on CSS custom properties. Without tokens.css, nothing renders correctly.
Never hardcode hex values
If an example shows a raw hex, it is a documentation bug — report it. Use the corresponding var(--token) instead. Run bash tokens/validate.sh to catch violations.
Run validate.sh before shipping
The linter catches raw hex, banned colours, unauthorized font weights, missing focus rings, and off-scale spacing. See Tooling for full docs.
Respect status badges
Stable = production-ready. Beta = API may change. Draft = do not use in production.

Requesting components and tokens

Before requesting anything new, check that it does not already exist under a different name.

1
Check existing components and token reference
Browse the component index and token reference. The need may already be served.
2
File an RFC
Use the RFC template in Governance. Include: problem, existing pattern check, token impact, a11y impact, TV impact.
3
DLS owner reviews
Reviewed against: is this truly shared? Does it justify the token count increase? Does it work on all 3 platforms?

Pre-ship checklist

Run through this before every feature ship. The full release checklist lives in Governance.

bash tokens/validate.sh path/to/screen.html exits 0
Focus ring visible on keyboard navigation (all interactive elements)
TV: tested with keyboard D-pad simulation (Tab, Arrow keys, Enter, Escape)
No raw hex values outside of tokens.css
Reduced-motion tested: no looping animations at rest
Placeholder/helper text uses --input-placeholder, not --text3
No blue or purple used as accent colour (JioGames is green-only)

Font loading

JioGames DLS uses JioType exclusively. Correct loading prevents FOUT (flash of unstyled text) and layout shift.

Required weights

WeightNameUsage
300LightLarge hero sub-copy, legal text.
500MediumBody text, captions, secondary labels.
700BoldHeadings, section titles, CTAs.
900BlackHero titles, stat numbers, display text.

Do not load 400 (Regular) or 600 (SemiBold) — DLS does not use them. Loading unused weights wastes ~60KB per variant.

Recommended @font-face

@font-face {
  font-family: 'JioType';
  src: url('/fonts/JioType-Light.woff2') format('woff2');
  font-weight: 300;
  font-style: normal;
  font-display: swap;
}
@font-face {
  font-family: 'JioType';
  src: url('/fonts/JioType-Medium.woff2') format('woff2');
  font-weight: 500;
  font-style: normal;
  font-display: swap;
}
@font-face {
  font-family: 'JioType';
  src: url('/fonts/JioType-Bold.woff2') format('woff2');
  font-weight: 700;
  font-style: normal;
  font-display: swap;
}
@font-face {
  font-family: 'JioType';
  src: url('/fonts/JioType-Black.woff2') format('woff2');
  font-weight: 900;
  font-style: normal;
  font-display: swap;
}

Preload critical weights

Preload Bold and Black — they render above-the-fold hero text. Light and Medium can load normally.

<link rel="preload" href="/fonts/JioType-Bold.woff2"
      as="font" type="font/woff2" crossorigin>
<link rel="preload" href="/fonts/JioType-Black.woff2"
      as="font" type="font/woff2" crossorigin>

Fallback stack

/* Already in tokens.css */
--jio-font: 'JioType', 'Outfit', -apple-system, BlinkMacSystemFont, sans-serif;

Outfit is the closest public fallback — similar proportions. Do not substitute Inter or Roboto.

font-display: block — never use
block hides text for up to 3s on slow connections. JioGames targets low-end Android — use swap always.
Hosting JioType externally
JioType is proprietary. Do not self-host on public CDNs or open-source repos. Serve from your own domain or Jio CDN only.
