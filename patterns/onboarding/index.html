Onboarding

Genre and platform preference collection for new users. Appears after OTP verification for States 1 and 3. Skippable at every step — JioGames never gates access on preference completion.

Mobile Beta

Flow map

1
Welcome screen Brief value prop. State 3 (subscriber bundled) shows green banner "Pass included with your plan". Step count shown (e.g. "Step 1 of 2").
2
Genre selection Grid of genre tiles (min 3 selections required). Progress bar advances. "Skip" always visible in top-right. Continue CTA disabled until ≥3 selected.
3
Platform selection 2 choices: Mobile only, or All Screens. No minimum. "Skip" still available. Single-select, not multi-select.
4
Done → Home Brief confirmation animation (500ms, respect prefers-reduced-motion). Navigate to home with personalised rail set. No explicit done screen — animate out directly to home.

Skip logic

Skip genre step
Platform step still shown. Preferences saved as empty array. Home uses non-personalised default rails.
Skip all (top-level)
Both steps skipped. Navigates directly to home. API call still sent with empty preferences (not omitted).
Skip platform step
Genre prefs saved. Platform defaults to Mobile. Same result as selecting Mobile explicitly.
Resume partial
If user kills app mid-flow: resume at last incomplete step on next open (within 7 days). After 7 days: skip, go to home.

Genre tile spec

Genre tiles are NOT a DLS component (no standalone page). They are a pattern-exclusive primitive.

PropertyValue
Size104px × 104px on 2-column grid, 8px gap
Border radiusvar(--r4)
BackgroundGame cover image (object-fit:cover) with 40% dark scrim
LabelGenre name, bottom-left, 12px, font-weight:700, white
Selected stateBorder: 2px solid var(--jio). Check icon top-right. Scrim lightens to 20%.
Unselected stateNo border. Scrim 40%.
Min selections3 required before Continue CTA enables
Count badgeNone visible — selection count tracked in sub-copy below grid ("3 selected")

Accessibility

  • ·Genre grid: role="group" with accessible name "Select genres". Each tile: role="checkbox" with game name + "selected/not selected" state.
  • ·Progress bar: role="progressbar", aria-valuenow, aria-valuemin="0", aria-valuemax="2"
  • ·Skip button always reachable before grid in tab order
  • ·Continue CTA: aria-disabled="true" (not disabled) until 3 genres selected — remains focusable
  • ·On step advance: focus moves to new step heading via tabIndex="-1" + JS focus()
  • ·Completion animation: prefers-reduced-motion: reduce → skip to home immediately
