Variants

Four patterns cover the full range of contextual use cases. Each is interactive below.

Basic

Fixed-height sheet with a list of actions. Single snap point. The most common variant — used for sharing, more options, and quick selections.

Snap points

Multi-height sheet that snaps between defined heights on drag. Supports peek (25%), half (55%), and full (92%). Click the buttons below to snap between heights.

Define snap heights as percentages of the frame. On release after drag, snap to nearest point based on position and velocity. Higher velocity snaps in direction of travel regardless of midpoint.

JS — snap point logic
const snaps = ['25%', '55%', '92%']; // of viewport height
let current = 0;

function snapTo(index) {
  current = index;
  sheet.style.height = snaps[index];
  updateIndicators();
}

// On drag end — snap by velocity
function onDragEnd(velocity, position) {
  if (velocity > 0.5) snapTo(Math.max(0, current - 1));  // fast down = lower snap
  else if (velocity < -0.5) snapTo(Math.min(2, current + 1)); // fast up = higher snap
  else snapTo(nearestSnap(position));
}

Anatomy

The component hierarchy. Required parts must always be present. Optional parts enhance specific use cases.

Drawer Required
Root context provider. Controls open/close state, snap point position, and animation. Sets role="dialog" and aria-modal="true".
└─ DrawerTrigger Required
The element that opens the drawer. Wraps any interactive element. Sets aria-haspopup="dialog" and aria-controls. Focus returns here on close.
└─ DrawerPortal Optional
Teleports drawer content to <body> to escape stacking context. Use when the triggering element is inside a transformed or overflow:hidden ancestor.
   └─ DrawerOverlay Required
Scrim layer behind the sheet. rgba(0,0,0,.45). Click dismisses the drawer. Opacity tracks drag position during swipe-dismiss. Blocks background interaction.
   └─ DrawerContent Required
The sheet container. background: var(--surface-2), border-radius: 20px 20px 0 0. Handles touch events for drag and snap. Receives focus on open.
      └─ DrawerHandle Required
40×4px pill, rgba(255,255,255,.2), centered at top. Communicates draggability. Keyboard-focusable — Space/Enter snaps to next height, Escape dismisses. Never omit.
      └─ DrawerHeader Optional
Contains title and optional description. flex-shrink:0 — does not scroll with body. Use a bottom border when the body is scrollable to signal more content below.
         ├─ DrawerTitle Optional
Labelled with id matched by aria-labelledby on DrawerContent. Font: 15–17px / 700. Max 4 words. Describes the action, not the component.
         └─ DrawerDescription Optional
Short supporting text. Linked via aria-describedby. Font: 13px / 400 / var(--text2). Max 2 lines. Not a substitute for button labels.
      └─ DrawerBody Required
Scrollable content area. flex:1; overflow-y:auto; min-height:0. Contains action rows, form fields, lists, or rich content. Scrolls independently of header and footer.
      └─ DrawerFooter Optional
Sticky footer for primary actions. flex-shrink:0. Always visible above the keyboard. Primary action on the right. Max 2 actions. Includes env(safe-area-inset-bottom) bottom padding.

Open / close — JS
const sheet  = document.querySelector('.bottom-sheet');
const overlay = document.querySelector('.sheet-overlay');

function openSheet() {
  sheet.classList.add('is-open');
  overlay.classList.add('is-visible');
  document.body.style.overflow = 'hidden';
  sheet.focus();
}

function closeSheet() {
  sheet.classList.remove('is-open');
  overlay.classList.remove('is-visible');
  document.body.style.overflow = '';
  document.querySelector('[data-sheet-trigger]')?.focus(); // restore focus
}

overlay.addEventListener('click', closeSheet);
document.addEventListener('keydown', e => { if (e.key === 'Escape') closeSheet(); });
sheet.querySelector('.sheet-handle').addEventListener('keydown', e => {
  if (e.key === 'Escape') closeSheet();
});

Do / Don't

Do
  • Use for actions related to the current context
  • Include a drag handle on every sheet — always
  • Use scrollable variant when content may overflow
  • Pin footer actions so they're always reachable
  • Dismiss on scrim tap, swipe down, or Escape
  • Return focus to trigger element on close
  • Add env(safe-area-inset-bottom) at bottom
  • Use snap points for variable-length content
Don't
  • Nest a sheet inside another sheet
  • Use on TV — bottom is outside safe zone
  • Use on desktop web — use side panel or dialog
  • Auto-open without a user trigger
  • Omit the drag handle — it signals draggability
  • Use for primary navigation or wayfinding
  • Put more than 6 items without scroll or snap
  • Use generic titles like "Options" or "Menu"

Platform guidance

📱 Mobile
  • Primary pattern — use freely
  • Touch drag with velocity-aware snap
  • Snap heights: peek / half / full
  • Corner radius 20px, handle 40×4px
  • Safe area bottom padding required
  • Background scales to 93% when sheet is open
💻 Web
  • Avoid on desktop viewports ≥768px
  • Switch to centered dialog at 768px+
  • Drag-to-dismiss not available on pointer
  • Escape key must always dismiss
  • Tab cycles focus within sheet (trap)
  • Click outside backdrop dismisses
📺 TV
  • Do not use — pattern not supported
  • Bottom of screen is outside TV safe zone
  • Use full-screen overlay instead
  • No touch gestures on TV
  • D-pad cannot interact with bottom-anchored drawers
  • Use focused panel or dialog for confirmations

Accessibility

ARIA role
Modal sheets: role="dialog" + aria-modal="true". Non-modal supplemental sheets: role="complementary". Both require aria-labelledby pointing to DrawerTitle.
Focus trap
Modal sheets trap focus inside DrawerContent. Tab cycles through interactive elements; Shift+Tab reverses. Focus must not escape to background content while the sheet is open.
Handle keyboard
DrawerHandle has tabindex="0". Space / Enter snaps to next height. Escape dismisses. Minimum touch target: 44×44px (extend handle invisibly).
Snap announce
Announce height changes via aria-live="polite". Example: "Panel expanded to half height". Keep announcements under 5 words and fire after transition ends.
Keyboard dismiss
Escape dismisses at any snap height. Restore focus to DrawerTrigger. On mobile, the system Back gesture must also dismiss.
Reduced motion
Wrap all transition and transform in @media (prefers-reduced-motion: no-preference). Under reduced motion, sheet appears instantly at target position with no animation.
