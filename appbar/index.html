App Bar

The app bar orients players within JioGames, providing navigation context, page title, and contextual actions.

The app bar sits at the top of every JioGames screen. It provides players with their location, a way back, and up to three contextual actions. On mobile it is scroll-aware — it can become transparent over hero images and condensed when the player scrolls past the hero.

  • Navigation context. Back button or close button. Only one navigation action — never both back and hamburger.
  • Title. Page title or logo. Sentence case, max 24 characters. Keep short.
  • Trailing actions. Up to 3 icon buttons, rightmost is the most important. No text labels.
  • Adaptive. Transparent over hero images; opaque when scrolled. Background transitions with var(--dur-default).

Best Practices

Follow these guidelines to keep navigation consistent and accessible across all JioGames surfaces.

Do
  • Use chevron-left ‹ for back (not ←)
  • Keep title under 24 characters
  • Use a maximum of 3 trailing icons
  • Apply safe-area-inset-top on mobile
  • Transition background on scroll for cinematic screens
  • Use aria-label on all icon buttons
Don't
  • Don't add text labels to trailing icon buttons
  • Don't exceed 3 trailing icons
  • Don't use hamburger + back simultaneously
  • Don't truncate title with ellipsis unless absolutely necessary

Content Guidance

Writing and icon selection rules that keep the app bar clear and purposeful.

Title writing
  • Sentence case only
  • Match the page or section name exactly
  • Max 24 characters (hard limit)
  • Avoid "JioGames" prefix — it's already contextually clear
  • "My Library" not "Library Page" or "Your Library"
Action icons
  • Rightmost = most important action (e.g., share)
  • Standard icons: share, bookmark, filter, more (vertical dots)
  • Never add text labels to trailing icon buttons
  • If you need more than 3, hide extras in "more" overflow

Platform Considerations

The app bar is a mobile-only component. The rules below apply to the JioGames mobile app.

Mobile
  • Scroll-aware transparency over hero images
  • safe-area-inset-top required for iOS notch
  • Back chevron with 44px touch target
  • Height var(--app-bar-h) — 64px
  • Cinematic mode for game detail screens

Accessibility

The app bar must be keyboard navigable and fully described to screen readers.

Roles & Markup
  • <header> element for main navigation bars
  • <nav> if it contains navigation links
  • aria-label="Page navigation" on the header
Keyboard
  • Back button is Tab-stop 1 on every screen
  • Trailing icons Tab 2–4 in document order
  • Escape key on mobile triggers back action
ARIA Labels
  • aria-label="Go back" on back button
  • aria-label="Share" on each trailing icon
  • aria-current="page" not needed on bar itself
Focus
  • 2px var(--jio) focus ring on all icon buttons
  • Skip-nav link before bar on web
  • TV: no programmatic focus on bar elements — use remote
