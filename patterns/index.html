Pattern rules

Patterns compose existing components
No pattern may introduce new visual primitives, new tokens, or new component variants without an RFC. Patterns are flow documentation only.
Platform differences are explicit
Every pattern documents Mobile, Web, and TV behaviour separately where they differ. Never assume mobile == web.
Accessibility is always specified
Every pattern includes focus management, screen reader announcements, and reduced-motion behaviour.
