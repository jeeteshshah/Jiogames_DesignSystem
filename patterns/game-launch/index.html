Game Launch

The sequence from tapping Play through loading, launch, and error handling. Covers free, pass-required, and unauthenticated flows.

Launch sequence

1
Button loading Play button enters loading state immediately on tap (spinner replaces label). Page remains visible.
2
Full-screen loader After 800ms: transition to full-screen loading overlay. Shows game cover art + title + progress bar. "Preparing game…" sub-copy.
3a
Launch success Game takes over full screen. OS navigation hidden. Back gesture / back button returns to game detail (not home).
3b
Launch failure Error dialog over full-screen loader. Primary: Retry. Secondary: Go back. Timeout error after 30s: specific "Taking too long" message.

Timeout handling

DurationTriggerAction
0–800msNormalButton spinner only. No overlay.
800msStill loadingTransition to full-screen loading overlay.
10sStill loadingSub-copy changes to "Still loading…". Retry option appears.
30sTimeoutError dialog: "Game is taking too long to start." Retry + Go back.

Platform differences

Mobile

  • Game launches in WebView or native wrapper
  • Back swipe → game detail page
  • Status bar hidden during play

Web

  • Game iframe or new tab (game-dependent)
  • Full-screen loader in overlay before iframe shows
  • Browser back = game detail (pushState managed)

TV

  • Loading screen uses TV focus ring on cancel button
  • Remote Back cancels launch at any point
  • Error dialog: D-pad navigable, OK = retry

Accessibility

  • ·Button loading: aria-busy="true" on button + visible spinner
  • ·Full-screen overlay: role="status" with live region for loading text updates
  • ·Error dialog: focus trap. Escape + explicit close. Announce error via role="alertdialog"
  • ·Progress bar on loader: role="progressbar" with indeterminate state (aria-valuenow omitted)
  • ·Reduced motion: skip animated progress bar; show static loading indicator instead
