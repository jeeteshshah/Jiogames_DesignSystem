Payment & Checkout

The complete flow for JioGames Pass subscription purchase — from entry point to confirmed or failed state.

Mobile Web Beta

Flow map

1
Entry User taps upgrade CTA on home rail, game detail, or locked content overlay. Entry source is logged for analytics.
2
Plan selection Pass cards shown (Mobile / Ultimate). One plan pre-selected based on entry context. Feature comparison table below cards. Coupon/offer badge if eligible.
3
Coupon / offer Optional step. Input field with apply CTA. Success shows discounted price in green badge. Failure shows inline error below field (not toast).
4
Payment method List of saved methods + add new. Selected method highlighted with jio-green border. Order summary pinned at bottom.
5
Processing Full-screen loading state. CTA enters loading state. Page interaction disabled. Skeleton not used here — payment is a point-in-time action, not a content load.
6a
Success Full-screen confirmation with plan name, start date, and "Start playing" CTA. Toast summary shown on return to home.
6b
Failure / retry Error dialog (not toast) with specific reason if available. Primary CTA: Try again. Secondary: Change payment method. Third attempt shows customer support link.

Entry points

Home upgrade rail
Combined pass upsell card in home rail. Non-dismissible for free users.
Locked game overlay
Tapping Play on a Premium-only game. Overlay shows lock icon + "Unlock with Pass" CTA.
Profile / account
Upgrade section in profile. For lapsed users: "Reactivate pass" entry.
Banner dismissal
Inline dismissible banner on home (State 8). "See plans" CTA opens checkout directly at plan selection.

Platform differences

Mobile

  • Full-screen sheets for each step
  • Back swipe navigates to previous step
  • Payment methods via native payment sheet
  • Order summary pinned above safe area bottom

Web

  • Modal dialog (max-width 640px) over blurred bg
  • Multi-step within dialog, no page navigation
  • Escape key: confirm-cancel dialog before closing
  • Sticky order summary in right column on ≥1024px

TV

  • TV checkout is typically QR code to mobile
  • Show QR + "Continue on phone" message
  • Poll for completion; auto-advance on success
  • Back always available via remote Back button

Accessibility

  • ·Multi-step flow: announce step progress with aria-live="polite" on step heading
  • ·Plan selection: radio group pattern with role="radiogroup"
  • ·Loading state: aria-busy="true" on the payment container
  • ·Success: focus moves to confirmation heading automatically
  • ·Error dialog: focus trapped inside. Escape + explicit close button both dismiss.
  • ·Price and plan differences must not rely on colour alone — use text labels alongside green/default badges
