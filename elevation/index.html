Shadow scale

Seven shadow levels cover every use case from flat page elements to top-of-stack TV cards. Use shadow tokens from tokens.css — not raw box-shadow values.

Level 0 no shadow
Level 1 --shadow-1
Level 2 --shadow-2
Level 3 --shadow-3
Level 4 --shadow-4
Glow --jio-glow
TV --shadow-tv
Level 0
Flat. Inline content, in-page elements.
Level 1
Subtle depth. Inline cards, list items.
Level 2
Standard cards. Game cards, content tiles.
Level 3
Floating panels. Dropdowns, tooltips.
Level 4
Modals and sheets. Bottom sheets, dialogs.
Glow
Brand highlight. Active cards, selected state.
TV
TV cards. Focused card on large screen.

Shadow tokens --shadow-1 through --shadow-4 are defined in tokens.css. Use these tokens, not raw box-shadow values.

.card           { box-shadow: var(--shadow-2); }
.card:hover     { box-shadow: var(--shadow-3); }
.card.active    { box-shadow: var(--jio-glow); }

Surface ladder

Six opaque surface tokens create the depth ramp. Each step is slightly lighter than the one below it — communicating that it sits on top of it.

--bg
Page background. Always the darkest surface.
#06080F
--card-bg
Default card surface. 1 level above page.
#111115
--surface-1
Elevated surface. Nav overlays.
#0E1119
--surface-2
Mid-elevation. Tooltips, popovers.
#161A24
--surface-3
High elevation. Modals, dialogs.
#1F2432
--surface-4
Highest surface. Keyboard, top overlays.
#2A3142

Elevation + surface pairing

Shadow and surface are always used together. Each layer has a canonical surface token and shadow token pair. Do not mix levels.

Layer Surface token Shadow token Typical component
0 — Page --bg none Page background
1 — Content --card-bg --shadow-2 Game cards, list items
2 — Float --surface-2 --shadow-3 Dropdowns, popovers
3 — Overlay --surface-3 --shadow-4 Bottom sheets, dialogs
4 — Top --surface-4 --shadow-4 On-screen keyboard

Active / selected states

Active and selected states use var(--jio-glow) instead of grey shadows. A green border at reduced opacity accompanies the glow. Never use grey box-shadows on selected elements.

Inactive Tomb Raider Action Adventure · 8.4GB box-shadow: var(--shadow-2)
Selected Tomb Raider Action Adventure · 8.4GB box-shadow: var(--jio-glow) 
/* Standard card — grey shadow */
.card {
  background:  var(--card-bg);
  box-shadow:  var(--shadow-2);
  border:      1px solid var(--border);
}

/* Active / selected card — green glow, no grey shadow */
.card.selected {
  box-shadow:  var(--jio-glow);
  border-color: rgba(0,168,89,.4);
}

Usage rules

Use shadow tokens (--shadow-1 through --shadow-4) — not raw box-shadow values.
Use --jio-glow for active, selected, and focused interactive elements.
Pair shadow level with the matching surface token from the ladder.
Use grey box-shadows on active or selected states — use --jio-glow.
Apply box-shadow to --bg (page background has no shadow).
Use white or coloured box-shadows — shadows are black at varying opacity only.
Use --shadow-4 on inline content — it is for modals and overlays only.
