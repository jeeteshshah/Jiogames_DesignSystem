Duration tokens

Duration tokens are defined in tokens.css and must be referenced via var(--dur-*) — never raw milliseconds in component CSS.

Token Value Purpose Typical use
--dur-instant 90ms Micro-feedback Icon state changes, toggle switches
--dur-fast 120ms Quick response Button press feedback, chip selection
--dur-default 200ms Standard UI Most transitions, fades, state changes
--dur-pop 280ms Pop-in content Cards appearing, dropdowns opening
--dur-error 380ms Error feedback Shake animations, validation errors
--dur-sheet 400ms Sheet slides Bottom sheet open / close
--dur-enter 420ms Screen enters New page appearing, overlay entering
--dur-screen 420ms Screen transitions Full navigation transitions
--dur-tv-enter 560ms TV focus / enter D-pad navigation, focus transitions
--dur-shimmer 1600ms Loading state Skeleton shimmer loop

Reduced motion

When prefers-reduced-motion: reduce is set, duration tokens are overridden in tokens.css. Animation properties themselves are not removed — only their durations, so no layout shifts occur.

@media (prefers-reduced-motion: reduce) {
  :root {
    --dur-instant: 0ms;
    --dur-fast:    0ms;
    --dur-default: 0ms;
    --dur-pop:     0ms;
    --dur-enter:   0ms;
    --dur-screen:  0ms;
    --dur-tv-enter:0ms;
    /* --dur-reduced-fade (100ms) and --dur-reduced-instant (1ms) */
    /* are intentionally NOT collapsed — they are reduced-motion values */
  }
}

/* Ambient loops must be explicitly stopped */
@media (prefers-reduced-motion: reduce) {
  .marquee-row, .hero-bg, .aurora {
    animation: none;
  }
  /* Shimmer: a zero-duration loop still flashes — remove it */
  .shimmer-wrap::after {
    animation: none;
    display:    none;
  }
}
Note: Shimmer animations must be explicitly removed (display:none or animation:none) for users with reduced motion. A zero-duration loop still flashes on every frame.

TV motion

TV is a 10-foot UI driven by D-pad. Motion must be larger, slightly slower, and completely restrained. No ambient loops, no press feedback, no sheets.

Focus transitions
All focus moves use --dur-tv-enter (560ms) + --spring. No instant cuts between focus states.
Card focus state
Focused card scales to scale(1.05) with --spring easing and mandatory --jio-glow box-shadow.
Focus ring timing
Focus ring appears in 150ms with --ease-out — intentionally faster than content transitions so navigation feels instant.
Screen transitions
Opacity crossfade only. No horizontal slide. TV spatial model does not follow the mobile forward/back convention.
Never on TV
No --spring-bounce — too playful for 10-foot UI. No ambient loops. No press/tap scale. No shimmer sweep. No backdrop-filter. No stagger sequences.
Content entrance
Use --dur-tv-enter with fade-up keyframe (32px translate). Stagger collapses — all items enter simultaneously on TV.
/* TV focus state */
.focusable:focus-visible {
  transform:  scale(1.05);
  box-shadow: var(--jio-glow);
  transition: transform var(--dur-fast) var(--spring),
              box-shadow var(--dur-fast) var(--spring);
  outline:    none;
}

/* TV content entrance — no stagger */
.tv-card {
  animation: fade-up var(--dur-tv-enter) var(--spring) both;
}

Usage rules

Use token pairs — always a duration token + an easing token together.
Use --spring for elements entering the screen.
Use --ease-out as the safe default when unsure about easing.
Use --dur-tv-enter + --spring for all TV focus moves and content entrance.
Use ad-hoc cubic-bezier values not in the token set — use var(--spring) etc.
Use transition: all — always list explicit properties.
Use --spring-bounce on TV — too playful for 10-foot UI.
Omit prefers-reduced-motion support for looping animations.
Animate pricing, payment totals, or subscription terms — critical information is always static.
