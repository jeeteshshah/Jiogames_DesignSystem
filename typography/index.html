Section 05

Usage rules

These rules are enforced by tokens/validate.sh. ERRs block shipping.

Use JioType for ALL text in JioGames interfaces — there are zero exceptions for interface text.
Use weight 900 for all headings, card titles, prices, and game names.
Apply negative letter-spacing (at least −0.4px) to all text at 20px or larger.
Use font-weight 500 as the default body text weight — it is the reading weight of JioType.
Use semantic type role classes (.text-hero, .text-body, etc.) — never hard-code font-size in components.
Use var(--text2) for body and caption text — never raw white or var(--text) for supporting copy.
Use any other typeface — Inter, Roboto, Helvetica, SF Pro, system fonts as primary are all banned.
Use font-weight 400, 600, or 800 — these weights do not exist in JioType and produce browser synthesis.
Apply positive letter-spacing to display text — all text ≥ 20px must use zero or negative tracking.
Use weight 300 (Light) for any interface text below 16px — Light is for large display only.
Use raw hex colors on text — always use var(--text), var(--text2), var(--text3), or var(--jio).
Use var(--text3) for critical information — it falls below WCAG AA at small sizes and is decorative-only.
